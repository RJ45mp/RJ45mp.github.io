portswigger-Authentication-上

0x00 前言

portswigger，一个关于burpsuite的靶场，burpsuite对于web安全的重要性不言而喻。

前面几天已经写了第一部分的Information disclosure信息泄露内容。

下面就第二部分的Authentication，即认证方面进行简单记录关卡过程。

0x01 Authentication

该部分一共有14关卡，都是关于认证过程的方式和如何攻击的内容。

身份验证机制是网站最常见的功能部分，

例如登录功能中的身份验证、敏感操作前的身份验证等等场景，

不同的身份验证机制有各自特定的漏洞。

一旦攻击者绕过身份验证到另一个用户的帐户中，

就可以访问被攻击帐户所拥有的数据和功能。

1、身份验证漏洞的起因

2、身份验证漏洞的类型

网站的身份验证系统通常由几种可能发生漏洞的不同机制组成。

在portswigger靶场中一共有三类常见漏洞：

0x02 第一关

Username enumeration via different responses

1、解释：该关卡是通过不同的响应信息来枚举账户。

2、过程：

进入靶场关卡、可以发现提供了home目录和账户登录功能

点击登录功能

随手输入一个弱账户admin/admin,可以看到返回了 Invalid username

从这个信息可以猜测，在该站点的登录功能过程中，首先是校验是否存在该用户，如果存在该账户，接着再校验账户的密码。那么接下来就说先爆破用户了

由于这个靶场是考察burpsuite的使用技巧，所以就不上自己的字典了，使用burpsuite自带的字典，把用户和密码字典的加载进来

在爆破过程中，点击数据包长度排序，过不久就get到有效的用户名Administrator，

同时也返回了关键信息 Invalid password

用户名get it，接着再继续爆破密码
