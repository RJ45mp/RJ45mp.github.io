xray-crawlergo-xray-server酱-实现自动化挖洞

Rj45's blog
2020-06-17

前言

这段时间，一直在挖SRC，感觉到自动化挖洞的舒服，

加上前几天撸到xray高级版两个月使用，还有参加某个厂商的内部众测，需要测试n多个子域名，等等原因，萌生了组装自动化的想法。

于是有了这篇文章，《xray-crawlergo-xray-server酱-实现自动化挖洞》。

架构

简单说说这个架构：

  1. xray高级版中有subdomain的收集功能，利用这个功能实现子域名的收集，然后保存到本地的targets.json文件。
  2. crawlergo读取本地的targets.json文件，对文件中的每一个子域名进行爬虫爬取url,并将url发送给下行的另一个xray。
  3. 另一个xray收到url，启用webscan功能进行漏洞扫描检测，使用webhook-output输出模式，将检测过程发送到本地特定端口地址。
  4. 利用python结合server酱监听本地的特定端口地址，过滤出检测到的漏洞信息，将漏洞信息发送到微信。

xray-subdomain

高级版的xray具有子域名收集的功能，注意一下扫描后输出为targets.json即可
