以下为sql基础内容。

什么是关系型数据库和非关系型数据库？

1：

关系型数据库是以关系模型来创建的数据库。
关系模型就是指二维表格模型。
一个关系型数据库就是由二维表及其之间的联系组成的一个数据组织。
常见的关系型数据库有：
Mysql、Oracle、PostgreSQL、MicrosoftSQLServer、MicrosoftAccess。

2：

非关系型数据库是以非关系模型来创建的数据库。
非关系模型有列模型、键值对模型、文档类模型。
常见的关系型数据库有：
mongoDB、redis、MemcacheDB、Hbase。

mysql的术语

数据库：一些关联表的集合。
数据表：数据库中的类Excel表格。
列：具有相同数据类型的数据集合。
行：每一行用来描述某条记录的具体信息。
数据：行的具体信息、每个数据必须与该列的数据类型相同。
主键：具有唯一性、一个数据表中只能包含一个主键。
外键：用来关联两个表。
复合键：将多个列作为一个索引键，用来复合索引。
索引：对数据库表中一列或多列数据进行排序的一个结构，类似书目录。

mysql的权限机制

权限与安全息息相关。
在平常时候，经常遇到一些情况：
进入数据库后，当前用户的权限非常低，没有多大作用，只能查询当前数据库或者没有导出文件的权限，不能导出一句话木马等等。

mysql的权限机制存储在mysql的user、db、table_priv、columns_priv、和procs_priv这几个系统表中。

mysql的权限机制分为3个等级，每个权限等级下有不同的权限划分：
1、作用于整个mysql的权限等级
2、作用于某个或多个指定数据库的权限等级
3、作用于某个或多个指定数据表的权限等级

1：

db
该表存放着数据库级别的权限

2:

user
该表存放用户账户信息以及权限等级1的权限。

3：

table_priv、columns_priv、和procs_priv
存放着表、列和存储过程的权限。
